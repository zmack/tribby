Ok, so what else ?

You can make your own visualization modules using tribby. Well, using actionscript, but tribby does the boring job of talking to github and translating their gibberish into sweet, sweet numbers for you to play with.

The data is in ContibutionEvent's data member, quite imaginatively. It's stored in the form of an array of objects. Each element is an object containing two items, namely 'own', which is the respective user's number of commits, and 'total', which is the total number of commits.

So to go through all the data you'd have to do something similar to this : 

          data.forEach( function(item:Object, index:uint, arr:Array):void {
            trace('Own commits: ' + item.own + '. Total commits: ' + item.total);
          });
